我想做一个抽奖转盘的后台管理系统，后端用 Go + Gin + SQLite，管理后台用 Vue3 + Element Plus。H5 抽奖页面先不做，只做后台管理这部分。

功能大概这样：

一、奖品管理

管理员可以在后台添加奖品，比如一等奖手机、二等奖充电宝这些。每个奖品要能设置：奖品名称、中奖概率（比如1%）、库存数量（还剩几个可以中）。还可以填个描述和图片链接，方便以后前端展示。奖品列表要能增删改查，还能启用或禁用。添加或编辑的时候，系统要检查一下所有启用的奖品概率加起来不能超过100%，剩下的就是未中奖的概率。

二、抽奖记录

所有用户抽奖的记录都要存下来，包括：抽奖人姓名、手机号、中了什么奖品、有没有中奖、领奖状态（待领取/已领取）。后台要能看到这些记录，支持按姓名或手机号搜索、按奖品筛选、按中奖状态筛选。每条待领取的记录旁边有个按钮，点一下就能标记为“已领取”。还要支持把记录导出成Excel。

三、统计看板

做一个简单的数据看板，展示：总抽奖次数、中奖次数、中奖率、待领奖数量。再用图表展示各个奖品中了多少个（饼图），以及最近7天每天的抽奖趋势（柱状图，显示每天抽了多少次、中了多少次）。

四、登录

后台需要登录，内置一个 admin 账号，密码 123456，用 JWT 做身份验证。

五、抽奖核心逻辑

抽奖的时候，根据奖品设置的概率随机返回中奖结果。如果某个奖品库存没了（比如手机只有3台，中完了），那这个奖品就不能再被抽中，它的概率要分摊到其他奖品或者未中奖里。写接口的时候要考虑并发，别出现超卖的情况。

六、数据库表

就三张表：奖品表、抽奖记录表、管理员表。用 SQLite 就行，启动时自动创建表，如果 admin 用户不存在就自动创建。

七、前端页面

· 登录页
· 奖品管理页（表格、增删改查弹窗、启用禁用开关）
· 抽奖记录页（表格、筛选搜索、标记领取按钮、导出按钮）
· 统计看板页（卡片 + 饼图 + 柱状图）

侧边栏菜单要有这三个入口，路由守卫拦截未登录访问。

大概就这么个需求，代码放 server 和 admin 文件夹，结构清晰点，能直接跑起来。
注意sqllite 现在没cgo 扩展，用原生的。
