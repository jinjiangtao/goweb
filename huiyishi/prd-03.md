会议室预订系统 - 增加用户登录和多端数据同步

在现有会议室系统基础上，增加用户登录注册功能，实现跨浏览器数据同步。

一、改造说明

原来 H5 页面用手机号 localStorage 存身份，现在改成正式登录。用户从后台添加，登录后数据按用户 ID 存储，换浏览器也能看到自己的预订记录。

二、后端改造

新增用户表：

· 用户表：id, username, password（bcrypt）, real_name, phone, status（启用/禁用）, created_at
· 用户与预订表关联：预订表的 phone 字段改为 user_id 外键

管理员后台增加“用户管理”模块：

· 菜单加一级菜单“用户管理”（仅管理员可见）
· 功能：增删改查用户、重置密码、启用/禁用账号
· 添加用户时填写：用户名、密码、真实姓名、手机号
· 用户列表展示：用户名、真实姓名、手机号、状态、创建时间、操作

登录接口改造：

· POST /api/login - 用户登录，验证用户名密码，返回 JWT token 和用户信息
· GET /api/user/info - 获取当前用户信息（需 token）
· 原有的 POST /api/bookings 需改为需 token 或同时支持 token（优先）和手机号（兼容）
· GET /api/bookings/my 改为根据当前登录用户的 user_id 查询
· DELETE /api/bookings/:id 改为需 token，验证当前用户是否有权取消（自己的预订或管理员）

权限区分：

· 普通用户：只能预订、查看/取消自己的预订
· 管理员：保留原有的后台管理功能（会议室管理、预订管理、统计看板、用户管理）

三、H5 前端改造

增加登录/注册页面：

· 登录页：用户名、密码登录，登录成功存储 token 和用户信息，跳转到首页
· 注册页：注册需要填写用户名、密码、真实姓名、手机号，调用注册接口。注册后需管理员在后台审核通过才能登录（或直接设为启用状态，看你需求）
· 退出登录：清除 token，回到登录页

预订页面改造：

· 只有登录后才能访问预订页面和我的预订页面
· 提交预订时自动带当前用户 ID
· 我的预订页面直接根据登录用户展示数据，不再需要手动输入手机号
· 换浏览器或换手机，登录同一个账号后数据自动同步

路由守卫：

· 未登录时访问任何页面都跳转到登录页
· 登录后不能访问登录页/注册页

四、跨浏览器数据同步效果

· 用户 A 在手机 Chrome 登录账号预订了会议室
· 同账号在手机 Safari 或电脑浏览器登录后，能看到刚才的预订记录
· 取消预订后，所有设备同步显示已取消状态

五、接口汇总

公开接口（无需登录）：

· POST /api/register - 注册
· POST /api/login - 登录

需登录（普通用户）：

· GET /api/rooms
· GET /api/rooms/:id/available
· POST /api/bookings（创建预订）
· GET /api/bookings/my（我的预订）
· DELETE /api/bookings/:id（取消自己的预订）
· GET /api/user/info

需管理员权限：

· 原有 /api/admin/* 接口
· 新增用户管理 CRUD 接口
